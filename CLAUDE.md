# CLAUDE.md — 项目配置

## 通用规则

- 始终使用中文进行对话，技术术语和代码标识符保持原文
- 代码注释使用中文
- 优先使用现代最佳实践和安全编码标准
- 修改代码前先理解上下文，不要盲目修改
- 修改后运行相关测试确保无回归
- Git 提交信息遵循 Conventional Commits 规范

## 代码风格

- 保持与周围代码一致的风格（注释密度、命名习惯、代码惯用法）
- 不要引入不必要的依赖
- 优先使用项目已有的工具和库
- 函数和类保持单一职责
- 错误处理要完善，避免空 catch

## 文件操作

- 创建新文件前检查是否已存在类似功能
- 修改文件前先读取了解完整上下文
- 避免一次性大规模重构，优先小步迭代
- 删除代码前确认无其他引用

## 测试

- 新功能应当有对应的测试
- Bug 修复应包含回归测试
- 运行测试时注意超时和隔离问题

## 安全

- 不在代码中硬编码密钥、令牌或密码
- 使用环境变量管理敏感配置
- 验证用户输入，防范注入攻击
- 注意依赖安全漏洞
